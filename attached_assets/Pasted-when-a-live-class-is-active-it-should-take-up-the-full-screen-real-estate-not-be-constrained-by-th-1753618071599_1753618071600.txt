The Solution: Context-Aware Layout
We need to implement a full-screen mode that hides the sidebar completely when a live class is active, and shows participants properly without requiring scrolling.

Here are the technical prompts to give the Replit AI agent:

Prompt 1: Create Full-Screen Live Class Mode

Modify the main App.jsx or Layout.jsx component to conditionally hide the sidebar when a live class is active. Add a state variable isInLiveClass that gets passed down to child components and controls the layout:

jsx
// In your main layout component
const [isInLiveClass, setIsInLiveClass] = useState(false);

return (
  <div className="app-container">
    {!isInLiveClass && (
      <div className="sidebar">
        {/* Your existing sidebar content */}
      </div>
    )}
    <div className={isInLiveClass ? "main-content-fullscreen" : "main-content-with-sidebar"}>
      {/* Pass setIsInLiveClass to your live class components */}
      {activeView === 'live' && <LiveClassView onLiveClassStart={() => setIsInLiveClass(true)} onLiveClassEnd={() => setIsInLiveClass(false)} />}
    </div>
  </div>
);

What This Achieves
Full-Screen Experience: When a live class starts, the sidebar disappears completely, giving 100% screen space to the video content

Optimal Participant View: The main video area gets maximum space while participants are shown in a dedicated sidebar that doesn't require scrolling

Professional Layout: Uses CSS Grid for a clean, structured layout similar to Zoom or Google Meet

Mobile Responsive: Adapts properly on smaller screens

Easy Navigation: Users can still exit full-screen mode via the call controls

User Experience Flow
Normal State: User sees sidebar with "Live Classes" and "Recordings"

Class Starts: Sidebar disappears, full screen is dedicated to video

Class Ends: Returns to normal layout with sidebar visible

Clean Transition: Smooth experience without jarring layout shifts
